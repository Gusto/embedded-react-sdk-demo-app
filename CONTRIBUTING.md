# Contributing

Thanks for your interest in contributing to the Gusto Embedded SDK Demo. This is
a reference application for partner developers, and we welcome issues and pull
requests that make it a clearer, more useful example.

## How to contribute

1. Fork the repository and create a topic branch off `main`.
2. Make your change. For setup and how to run the two services locally, see the
   [README](./README.md).
3. Run the frontend linter before opening a PR:

   ```bash
   cd frontend && npm run lint
   ```

4. Open a pull request describing the change and the motivation behind it.

## Reporting issues

Please use GitHub issues for bug reports and feature suggestions. Do not include
secrets, access tokens, real company/employee identifiers, or any other
non-public data in issues or pull requests.
